# Frontend 개선 로드맵

## P0 — 즉시 개선 (필수)

### UI 디자인 리뉴얼
- [ ] 키워드 카드 디자인 개선 — 현재 카드 레이아웃을 더 직관적이고 시각적으로 매력적인 형태로 변경
- [ ] 내러티브 뷰 타이포그래피 — 본문 가독성 향상 (줄 높이, 폰트 크기, 문단 간격)
- [ ] 차트 시각화 통일 — Plotly/Chart.js 혼용 정리, 차트 스타일 가이드 확립
- [ ] 온보딩 플로우 — 첫 사용자 경험 개선 (이미지/애니메이션 추가)

### 반응형 대응
- [ ] 480px 이상 화면 대응 — 태블릿/데스크탑에서의 레이아웃 보완
- [ ] BottomNav + AppHeader — 화면 크기별 네비게이션 전환 (데스크탑: 사이드/탑바)
- [ ] 모달/바텀시트 — 화면 크기별 적절한 UI 패턴 적용

### 핵심 UX 버그
- [ ] SSE 스트리밍 안정성 — 튜터 응답 중 연결 끊김 시 재연결 로직
- [ ] 로딩 상태 일관성 — PenguinLoading 적용 누락 페이지 점검
- [ ] 에러 핸들링 — API 실패 시 사용자 친화적 에러 메시지

## P1 — 중기 개선 (품질)

### 접근성 (Accessibility)
- [ ] 시맨틱 HTML — `<header>`, `<main>`, `<nav>`, `<article>` 적용
- [ ] ARIA 속성 — 모달, 바텀시트, FAB에 `aria-label`, `role` 추가
- [ ] 키보드 네비게이션 — Tab 순서, Focus trap (모달)
- [ ] 색상 대비 — WCAG AA 기준 충족 (특히 다크모드)

### 성능 최적화
- [ ] 번들 사이즈 분석 — Plotly.js 등 대형 라이브러리 lazy import
- [ ] 이미지 최적화 — WebP 변환, lazy loading
- [ ] API 캐싱 — 키워드/케이스 데이터 SWR 또는 React Query 도입 검토
- [ ] 메모이제이션 — 불필요한 리렌더링 방지 (React.memo, useMemo)

### PWA 강화
- [ ] 오프라인 지원 — Service Worker 캐시 전략 고도화
- [ ] 푸시 알림 — 새 브리핑 도착 시 알림
- [ ] 설치 유도 — 커스텀 설치 프롬프트 UI

## P2 — 장기 개선 (확장)

### Storybook 도입
- [ ] 공통 컴포넌트 문서화 — `common/` 컴포넌트를 Storybook으로 카탈로그
- [ ] 디자인 시스템 — 색상, 타이포그래피, 간격, 컴포넌트 가이드 통합
- [ ] 비주얼 리그레션 테스트 — Chromatic 연동

### 테스트 확충
- [ ] 유닛 테스트 — Vitest 도입, 핵심 유틸/훅 테스트
- [ ] 컴포넌트 테스트 — React Testing Library로 주요 컴포넌트 테스트
- [ ] E2E 테스트 강화 — Playwright 시나리오 확대 (현재 기본 시나리오만 커버)

### 국제화 (i18n) 기반
- [ ] 텍스트 외부화 — 하드코딩된 한글 문자열을 키 기반으로 분리
- [ ] i18n 프레임워크 — react-intl 또는 i18next 도입 기반 마련
