# 손영진 — 프로젝트 기여 보고서 (STAR)

> 역할: 팀장 (기획, React UI)
> 기간: 2026년 2월
> 기술 스택: React 19, Vite, Tailwind CSS, Plotly, React Router

---

## Situation (상황)

- Adelie Investment는 AI 기반 금융 교육 플랫폼으로, 현재 한국 주식 시장 이벤트와 역사적 사례를 매칭하여 투자 교육을 제공하는 모바일 퍼스트(max-width: 480px) 서비스이다.
- 프로젝트 초기 프론트엔드는 빈 상태에서 시작하였으며, Figma 디자인 시안을 기반으로 React 19 + Vite 환경에서 전체 UI를 구축해야 했다.
- 팀장으로서 기획 방향 수립과 프론트엔드 전체 아키텍처 설계를 동시에 진행해야 하는 상황이었다.
- 백엔드 API 및 데이터 파이프라인과의 연동 인터페이스가 확정되지 않은 상태에서 병렬 개발이 필요했다.

## Task (과제)

- React 19 + Vite 기반 SPA 프론트엔드 전체 아키텍처 설계 및 구현
- Figma 디자인 시안을 컴포넌트 구조로 충실히 재구현
- 랜딩, 홈, 내러티브(브리핑), 검색, 종목 상세 등 핵심 페이지 구현
- AI 튜터 챗봇(ChatFAB) UI 통합 및 페이지별 맞춤 프롬프트 연동
- 차트 렌더링 안정화 및 데이터 시각화 UX 최적화
- 팀원 간 프론트-백엔드 인터페이스 조율 및 API 연동 총괄

## Action (행동)

### 주요 구현 사항

#### 페이지 및 레이아웃 구현

- **Figma 컴포넌트 구조 재구현**: 디자인 시안의 컴포넌트 계층 구조를 React 컴포넌트로 1:1 매핑하여 구현하였다.
  - `common/` — 재사용 가능한 공통 컴포넌트
  - `domain/` — 비즈니스 로직 컴포넌트
  - `layout/` — AppHeader, BottomNav, ChatFAB
  - `charts/` — Plotly 기반 시각화 컴포넌트
  - `tutor/` — AI 튜터 채팅 UI
  - `trading/` — 주식 시뮬레이션 컴포넌트
- **랜딩 페이지 5페이지 자동 전환**: 비로그인 사용자를 위한 랜딩 페이지에 스와이프 기반 자동 전환 기능을 구현하여 서비스 소개 플로우를 완성하였다.
- **랜딩-홈-콘텐츠 흐름 및 하단탭 UI**: 사용자의 메인 네비게이션 경로를 `BottomNav` 컴포넌트와 함께 설계하여 모바일 앱 수준의 탐색 경험을 제공하였다.

#### 내러티브 및 차트 안정화

- **내러티브 UI 재구성**: 브리핑 내러티브 페이지의 구조를 재설계하였다.
  - 6페이지 내러티브 콘텐츠의 스와이프 네비게이션 구현
  - 페이지별 차트/텍스트 레이아웃 최적화
- **Plotly 차트 렌더링 안정화**: Plotly 기반 차트 컴포넌트의 렌더링 타이밍 이슈를 해결하여 안정적인 데이터 시각화를 달성하였다.
- **종목 상세 바텀시트 차트 라인차트 안정화**: 종목 상세 화면의 바텀시트에서 표시되는 주가 차트를 라인차트로 전환하고, 데이터 로딩 및 렌더링 안정성을 확보하였다.
- **chart 생성 재시도 및 LLM 폴백 안정화**: 차트 데이터 생성 실패 시 재시도 로직과 LLM 프로바이더 폴백 메커니즘을 프론트엔드 에러 핸들링과 연동하여 안정화하였다.

#### AI 튜터 및 사용자 경험

- **ChatFAB 활성화 및 페이지별 맞춤 프롬프트**: 전역 AI 튜터 채팅 버튼(ChatFAB)을 구현하였다.
  - `TutorContext`를 통해 현재 페이지 컨텍스트에 맞는 프롬프트가 자동으로 설정
  - 내러티브 페이지에서는 해당 브리핑 내용을 컨텍스트로 전달
  - 홈 페이지에서는 오늘의 키워드 기반 질문을 유도
- **브리핑 종목 저장 중복 방지 로직**: 사용자가 동일 종목을 포트폴리오에 중복 저장하는 문제를 프론트엔드 레벨에서 검증 로직을 추가하여 해결하였다.
- **검색 및 차트 안정화**: 종목 검색 기능의 디바운싱 및 빈 결과 처리를 개선하고, 차트 컴포넌트의 리사이즈 대응 및 에러 바운더리를 추가하였다.

#### 파이프라인 연동 및 품질 개선

- **E2E 내러티브 콘텐츠 정제**: 파이프라인 → API → 프론트엔드를 관통하는 내러티브 콘텐츠의 표시 품질을 종단 간 개선하였다.
- **narrative spine 및 openai-first 생성 정책**: 내러티브 구조(spine)를 정의하고, OpenAI를 우선 프로바이더로 하는 생성 정책을 프론트엔드 렌더링 로직에 반영하였다.
- **validate_interface2 검증 입력 축소**: 파이프라인 검증 단계의 타임아웃 문제를 입력 데이터 축소를 통해 완화하였다.

### 기술적 의사결정

- **React 19 + Vite 선택**: 최신 React 기능(Suspense, Lazy Loading)과 Vite의 빠른 HMR을 활용하여 개발 생산성을 극대화하였다.
- **Tailwind CSS + CSS 변수 기반 테마**: 다크모드 지원을 `class` 전략으로 구현하고, CSS 변수를 통해 브랜드 컬러(`#FF6B00`)를 일관되게 관리하였다.
- **Context 기반 전역 상태 관리**: 외부 상태 관리 라이브러리(Redux, Zustand 등) 없이 React Context 계층 구조만으로 효율적인 전역 상태 관리를 달성하였다.
  - `ThemeProvider > UserProvider > PortfolioProvider > TutorProvider > TermProvider`
- **도메인 분리 API 레이어**: `src/api/` 디렉토리에 도메인별 API 파일을 분리하고, `fetchJson`/`postJson` 유틸을 통한 일관된 API 호출 패턴을 확립하였다.
- **React.lazy 기반 코드 스플리팅**: 페이지 단위 lazy loading을 적용하여 초기 번들 사이즈를 최소화하고 모바일 환경에서의 로딩 속도를 개선하였다.
