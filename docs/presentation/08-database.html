Q1 데이터베이스 아키텍처 개요 (PostgreSQL + pgvector + Redis + MinIO) +

Adelie Investment는 3가지 스토리지 시스템을 조합하여 사용합니다. 각각 역할이 명확히 분리되어 있으며, 모든 운영 데이터는 PostgreSQL에, 캐시 및 레이트리밋은 Redis에, 대용량 파일은 MinIO에 저장됩니다.

1. PostgreSQL 16 (pgvector + pg_trgm)

  • 역할: 주요 OLTP 데이터베이스 -- 21개 이상 테이블, 모든 비즈니스 데이터 저장
  • 드라이버: asyncpg (SQLAlchemy async) -- FastAPI 비동기 I/O에 최적화
  • pgvector: Vector(1536) 컬럼으로 OpenAI 임베딩 저장, ivfflat 인덱스로 코사인 유사도 검색
  • pg_trgm: GIN 인덱스 기반 한글 퍼지 텍스트 검색 (용어집, 역사 사례)
  • DB명: narrative_invest

2. Redis 7

  • 역할: 캐싱 레이어 + 레이트리밋 엔진
  • 캐싱: AI Tutor 용어 설명 (24h TTL), 용어집 (24h), 사용자 설정 (2h), 채팅 메시지 (1h)
  • 레이트리밋: Lua 스크립트 기반 슬라이딩 윈도우 (100 req/min per IP, ZSET 사용)
  • 연결 풀: max_connections=50, redis.asyncio 비동기 클라이언트

3. MinIO

  • 역할: S3 호환 오브젝트 스토리지 -- 증권사 리서치 리포트 PDF 저장
  • 사용: broker_reports.minio_path 컬럼에 저장 경로 참조
  • 배포: 프로덕션 환경에서만 운용 (docker-compose.prod.yml)

Q3 SQLAlchemy 비동기 ORM 패턴 (Mapped[T] + mapped_column) +

Adelie Investment는 SQLAlchemy 2.0+ 비동기 스타일을 사용합니다. 레거시 Column() 대신 mapped_column()과 Python 타입 어노테이션(Mapped[T])을 사용하여 타입 안전성과 IDE 자동완성을 확보합니다.

핵심 패턴

  • Mapped[int], Mapped[str], Mapped[datetime]: Python 타입 힌트로 컬럼 타입 추론
  • Mapped[Optional[T]]: nullable 컬럼 표현
  • mapped_column(): Column() 대체, primary_key, ForeignKey, Index 등 설정
  • relationship(back_populates=...): 양방향 관계 정의
  • DeclarativeBase: app.core.database.Base 클래스에서 상속

모델 코드 예시 (User 모델)

fastapi/app/models/user.py 
from sqlalchemy import String, Boolean, Time, ForeignKey, Index
from sqlalchemy.orm import Mapped, mapped_column, relationship

from app.core.database import Base


class User(Base):
    """User model."""

    __tablename__ = "users"

    id: Mapped[int] = mapped_column(primary_key=True)
    email: Mapped[str] = mapped_column(String(255), unique=True, nullable=False)
    username: Mapped[str] = mapped_column(String(50), unique=True, nullable=False)
    password_hash: Mapped[str] = mapped_column(String(255), nullable=False)
    difficulty_level: Mapped[str] = mapped_column(
        String(20), default="beginner",
        comment="beginner, elementary, intermediate"
    )
    created_at: Mapped[datetime] = mapped_column(default=datetime.utcnow)
    last_login_at: Mapped[Optional[datetime]] = mapped_column(nullable=True)

    # Relationships (양방향)
    settings: Mapped[Optional["UserSettings"]] = relationship(
        back_populates="user", uselist=False
    )
    tutor_sessions: Mapped[list["TutorSession"]] = relationship(
        back_populates="user"
    )
    portfolios: Mapped[list["UserPortfolio"]] = relationship(
        back_populates="user"
    )

    __table_args__ = (
        Index("ix_users_email", "email"),
        Index("ix_users_username", "username"),
    )

1:NMapped[list["Child"]]User ↔ TutorSession

Q4 커넥션 풀링과 성능 최적화 +

FastAPI 비동기 환경에서 DB 커넥션 관리는 성능에 직접적인 영향을 미칩니다. Adelie Investment는 SQLAlchemy AsyncEngine의 커넥션 풀을 세밀하게 튜닝하고, Prometheus 메트릭으로 DB 쿼리를 모니터링합니다.

커넥션 풀 설정

fastapi/app/core/database.py 
engine = create_async_engine(
    settings.DATABASE_URL,
    echo=settings.DEBUG,
    pool_pre_ping=True,      # stale 커넥션 감지
    pool_size=50,             # 기본 풀 크기
    max_overflow=100,         # 추가 허용 커넥션
    pool_timeout=30,          # 풀 대기 타임아웃 (초)
    pool_recycle=1800,        # 커넥션 재활용 주기 (30분)
)
파라미터값설명
pool_size50풀에 유지하는 기본 커넥션 수
max_overflow100pool_size 초과 시 추가 생성 가능한 수 (최대 동시 150)
pool_timeout30초풀에서 커넥션 획득 대기 최대 시간
pool_recycle1800초 (30분)커넥션 수명 -- DB 서버의 idle timeout 방지
pool_pre_pingTrue사용 전 PING으로 stale 커넥션 감지 후 재연결

비동기 세션 설정

AsyncSessionLocal = async_sessionmaker(
    bind=engine,
    class_=AsyncSession,
    expire_on_commit=False,   # 커밋 후에도 객체 속성 유지
    autocommit=False,         # 수동 커밋
    autoflush=False,          # 수동 플러시
)
  • expire_on_commit=False: 커밋 후 SELECT 재발행 방지 -- 응답 직렬화 시 lazy loading N+1 문제 차단
  • autoflush=False: 쿼리 전 불필요한 자동 INSERT/UPDATE 방지

Prometheus 메트릭 -- DB 쿼리 카운터

SQLAlchemy의 after_cursor_execute 이벤트를 후킹하여 모든 쿼리를 operation(select/insert/update/delete)과 result(success/fail)로 분류하여 카운트합니다.

@event.listens_for(engine.sync_engine, "after_cursor_execute")
def _after_cursor_execute(conn, cursor, statement, parameters, context, executemany):
    DB_QUERY_TOTAL.labels(_extract_db_operation(statement), "success").inc()

@event.listens_for(engine.sync_engine, "handle_error")
def _handle_db_error(exception_context):
    statement = getattr(exception_context, "statement", "") or ""
    DB_QUERY_TOTAL.labels(_extract_db_operation(statement), "fail").inc()

Grafana 대시보드에서 db_query_total 메트릭으로 쿼리 처리량 및 오류율을 실시간 모니터링합니다.

세션 라이프사이클 (Dependency Injection)

async def get_db() -> AsyncGenerator[AsyncSession, None]:
    """Dependency for getting async database session."""
    async with AsyncSessionLocal() as session:
        try:
            yield session
        finally:
            await session.close()

FastAPI의 Depends(get_db)로 라우트 핸들러에 주입되며, 요청 종료 시 자동으로 세션이 반환됩니다.

Q5 pgvector 벡터 검색 구현 +

Adelie Investment는 pgvector 확장을 사용하여 역사적 사례(historical_cases)의 의미적 유사도 검색을 구현합니다. OpenAI의 text-embedding-3-small 모델로 생성된 1536차원 임베딩 벡터를 저장하고, 코사인 거리 기반 ivfflat 인덱스로 검색합니다.

1. 모델 정의 -- Vector(1536) 컬럼

fastapi/app/models/historical_case.py 
try:
    from pgvector.sqlalchemy import Vector
except ImportError:
    Vector = None


class HistoricalCase(Base):
    __tablename__ = "historical_cases"

    id: Mapped[int] = mapped_column(primary_key=True)
    title: Mapped[str] = mapped_column(String(200), nullable=False)
    summary: Mapped[str] = mapped_column(Text, nullable=False)
    keywords: Mapped[Optional[dict]] = mapped_column(JSONB)
    embedding: Mapped[Optional[list]] = mapped_column(
        Vector(1536) if Vector else None,
        nullable=True,
        comment="OpenAI text-embedding-3-small 벡터"
    )

pgvector 미설치 폴백: try/except ImportError로 pgvector가 없는 환경(로컬 테스트)에서도 모델 로딩이 실패하지 않습니다.

2. 마이그레이션 -- 확장 활성화 + 인덱스 생성

database/alembic/versions/20260211_pg_trgm_pgvector_mv.py 
# pgvector 확장 활성화
op.execute("CREATE EXTENSION IF NOT EXISTS vector;")

# embedding 컬럼 추가
op.execute("""
    ALTER TABLE historical_cases
        ADD COLUMN IF NOT EXISTS embedding vector(1536);
""")

# ivfflat 인덱스 생성 (코사인 거리)
op.execute("""
    CREATE INDEX IF NOT EXISTS ix_historical_cases_embedding
        ON historical_cases USING ivfflat (embedding vector_cosine_ops)
        WITH (lists = 10);
""")

3. 벡터 검색 활용

항목상세
임베딩 모델OpenAI text-embedding-3-small (1536차원)
인덱스 타입ivfflat (Inverted File + Flat)
거리 함수vector_cosine_ops (코사인 유사도)
lists 파라미터10 (클러스터 수 -- 데이터 규모에 비례하여 설정)
사용 사례현재 시장 이벤트와 유사한 역사적 사례 자동 매칭

검색 쿼리 예시

-- 코사인 유사도 기반 상위 5개 유사 사례 검색
SELECT id, title, summary,
       1 - (embedding <=> $1::vector) AS similarity
FROM historical_cases
WHERE embedding IS NOT NULL
ORDER BY embedding <=> $1::vector
LIMIT 5;

<=>는 pgvector의 코사인 거리 연산자이며, 1 - distance로 유사도(0~1)를 계산합니다.

Q6 pg_trgm 퍼지 텍스트 검색 +

pg_trgm(trigram) 확장은 텍스트를 3글자 단위로 분해하여 유사도를 계산합니다. 한글 검색에서 정확한 키워드를 모르더라도 부분 일치나 오타를 허용하는 퍼지 검색이 가능합니다.

Materialized View -- 키워드 빈도 집계

daily_briefings의 top_keywords JSONB를 분해하여 키워드별 출현 빈도와 최근 등장일을 집계하는 Materialized View입니다.

CREATE MATERIALIZED VIEW IF NOT EXISTS mv_keyword_frequency AS
SELECT
    kw->>'title' AS keyword,
    COUNT(*) AS frequency,
    MAX(b.briefing_date) AS last_seen
FROM daily_briefings b,
     jsonb_array_elements(b.top_keywords->'keywords') AS kw
WHERE b.top_keywords IS NOT NULL
GROUP BY kw->>'title'
ORDER BY frequency DESC;

이 뷰에는 ix_mv_keyword_frequency_keyword UNIQUE 인덱스가 설정되어 있어 REFRESH MATERIALIZED VIEW CONCURRENTLY가 가능합니다.

퍼지 검색 쿼리 예시

-- "삼성전" 으로 "삼성전자" 검색 (유사도 0.3 이상)
SELECT term, definition_short,
       similarity(term, '삼성전') AS sim
FROM glossary
WHERE term % '삼성전'
ORDER BY sim DESC
LIMIT 10;

Q7 Redis 캐싱 전략과 패턴 +

Adelie Investment의 Redis 캐싱은 Cache-Aside (Lazy Loading) 패턴을 기반으로 합니다. 모든 캐시 연산은 비동기이며, Redis 장애 시에도 서비스가 정상 동작하도록 Graceful Fallback이 적용되어 있습니다.

캐시 타입별 TTL

fastapi/app/services/redis_cache.py 
TTL_TERM_EXPLANATION = 60 * 60 * 24  # 24시간
TTL_GLOSSARY = 60 * 60 * 24          # 24시간

Graceful Fallback 패턴

모든 Redis 연산은 _is_available() 체크 후 수행되며, 예외 발생 시 None 또는 False를 반환하여 DB 직접 조회로 폴백합니다.

async def get_term_explanation(self, term, difficulty="beginner"):
    if not self._is_available():
        return None                    # Redis 미연결 시 캐시 스킵
    key = f"term:{difficulty}:{term.lower()}"
    try:
        data = await self._client.get(key)
        CACHE_HIT_TOTAL.labels("term_explanation",
                               "true" if data else "false").inc()
        return data
    except Exception as e:
        logger.warning(f"Redis get_term_explanation error: {e}")
        return None                    # 오류 시 DB 폴백

async def invalidate_pipeline_caches(self) -> int:
    # api:keywords:today:* 패턴 삭제
    cursor = b"0"
    while True:

            cursor=cursor, match="api:keywords:today:*", count=100

Q8 Rate Limiting (Redis Lua Script) +

Adelie Investment는 슬라이딩 윈도우(Sliding Window) 알고리즘을 Redis ZSET + Lua 스크립트로 구현하여 IP 기반 레이트리밋을 적용합니다. 원자적(atomic) Lua 스크립트로 race condition 없이 정확한 카운팅을 보장합니다.

레이트리밋 설정

항목값
제한100 requests / 분 (per IP)
윈도우60,000ms (1분)
제외 경로/metrics, /docs, /redoc, /openapi.json, /
Redis 장애 시레이트리밋 미적용 (graceful fallback)

Lua 스크립트 (원자적 슬라이딩 윈도우)

fastapi/app/middleware/rate_limit.py 
SLIDING_WINDOW_LUA = """
local key = KEYS[1]
local now = tonumber(ARGV[1])
local window = tonumber(ARGV[2])
local limit = tonumber(ARGV[3])

-- 윈도우 밖 오래된 요소 제거
redis.call('ZREMRANGEBYSCORE', key, 0, now - window)
-- 현재 윈도우 내 요청 수
local count = redis.call('ZCARD', key)

if count < limit then
    -- 허용: 현재 timestamp를 score와 member로 추가
    redis.call('ZADD', key, now, now)
    redis.call('EXPIRE', key, math.ceil(window / 1000))
    return {1, limit - count - 1, 0}  -- {allowed, remaining, 0}
else
    -- 거부: 가장 오래된 요청의 만료 시점 계산
    local oldest = redis.call('ZRANGE', key, 0, 0, 'WITHSCORES')
    local reset = oldest[2] and (tonumber(oldest[2]) + window)
                             or (now + window)
    return {0, 0, reset}  -- {denied, 0, reset_timestamp}
end
"""

동작 원리

  1. ZREMRANGEBYSCORE: 현재 시간 - 윈도우(60초) 이전의 모든 엔트리 삭제
  2. ZCARD: 남은 엔트리 수 = 현재 윈도우 내 요청 수
  3. limit 미만이면 현재 timestamp를 ZADD하고 허용
  4. limit 이상이면 거부 + 가장 오래된 요청의 만료 시점(reset)을 반환

응답 헤더

429 (Too Many Requests) 응답 시 다음 헤더를 포함합니다:

headers = {
    "X-RateLimit-Limit": "100",
    "X-RateLimit-Remaining": "0",
    "X-RateLimit-Reset": "1709234567",     # Unix timestamp
    "Retry-After": "23",                    # 초 단위 대기 시간
}

미들웨어 구현

class RateLimitMiddleware(BaseHTTPMiddleware):
    async def dispatch(self, request, call_next):
        if request.url.path in _EXEMPT_PATHS:
            return await call_next(request)

        client_ip = request.client.host if request.client else "unknown"
        try:
            cache = await get_redis_cache()
            if cache.client:
                now_ms = int(time.time() * 1000)
                k = key_rate_limit("ip", client_ip)
                result = await cache.client.eval(
                    SLIDING_WINDOW_LUA, 1, k,
                    now_ms, WINDOW_MS, RATE_LIMIT
                )
                allowed, remaining, reset_ms = result
                if not allowed:
                    # 429 응답 반환
                    ...
        except Exception as e:
            # Redis 장애 시 레이트리밋 미적용
            logger.warning(f"RateLimit Redis error (fallback=allow): {e}")
