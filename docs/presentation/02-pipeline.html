Q4 뉴스/리서치 크롤링은 어떻게 동작하나? (crawl_news_node, crawl_research_node 병렬) +

뉴스와 리서치 크롤링은 collect_data_parallel wrapper 노드에서 asyncio.gather로 병렬 실행됩니다.

datapipeline/graph.py:125-141 
async def collect_data_parallel_node(state: dict) -> dict:
    """crawl_news + crawl_research 병렬 실행."""
    news_result, research_result = await asyncio.gather(
        asyncio.to_thread(crawl_news_node, state),
        asyncio.to_thread(crawl_research_node, state),
    )
    merged = {}
    merged.update(news_result)
    merged.update(research_result)
    return merged

def crawl_research(target_date, summarize=True, ...):
    with ThreadPoolExecutor(max_workers=2) as ex:
        f_ind = ex.submit(_iter_pages, INDUSTRY_LIST_URL, "industry", target_date)
        f_eco = ex.submit(_iter_pages, ECONOMY_LIST_URL, "economy", target_date)
        industry_items = f_ind.result()
        economy_items = f_eco.result()
    all_items = industry_items + economy_items
    # PDF 요약 (ThreadPoolExecutor, max_workers=4)
    with ThreadPoolExecutor(max_workers=max_workers) as ex:
        futures = {ex.submit(_summarize_one, item): item for item in all_items}
        for fut in as_completed(futures):
            results.append(fut.result())

Q5 종목 스크리닝 로직은? (screen_stocks, 5일 수익률 5%+, 거래량 비율 1.5x+, TOP 20) +

def _scan_stock(closes, volumes):
    signals = []
    price_now = closes.iloc[-1]

    # 거래량 스파이크: 당일 거래량 / 20일 평균 >= 1.5x
    vol_ratio = vol_last / vol_avg if vol_avg > 0 else 0
    if vol_ratio >= VOLUME_RATIO_MIN:
        signals.append({"signal": "volume_spike", ...})

    # 단기 급등/급락: 5일 수익률 +/-5%
    short_ret = (price_now - price_before) / price_before * 100
    if short_ret >= SHORT_TERM_RETURN_MIN:
        signals.append({"signal": "short_surge", ...})
    elif short_ret <= -SHORT_TERM_RETURN_MIN:
        signals.append({"signal": "short_drop", ...})

    # 중기 상승: 6개월 형성 + 1개월 skip, 5% 이상
    mid_ret = (p_end - p_start) / p_start * 100
    if mid_ret >= MID_TERM_RETURN_MIN:
        signals.append({"signal": "mid_term_up", ...})

Q6 LLM 큐레이션(curate_topics)은 어떻게 동작하나? +

GPT-5.2 + OpenAI Responses API의 web_search 도구를 활용하여 투자 테마를 큐레이션합니다. Strict JSON Schema 검증으로 출력 형식을 강제합니다.

  • 뉴스 요약 (GPT-5 mini Map/Reduce 결과)
  • 리포트 요약 (GPT-5 mini Map/Reduce 결과)

Q12 멀티 토픽 실행은 어떻게 동작하나? +

데이터 수집 모드에서는 첫 번째 토픽은 전체 파이프라인을 실행하고, 2번째 이후는 데이터 수집을 재사용하여 Interface 2/3만 실행합니다. 기본 토픽 수는 3개입니다.

def _build_full_content(narrative: dict, final: dict) -> str:
    parts = []
    # 내러티브 섹션
    for section_name in ["background", "concept_explain", "history", ...]:
        section = body.get(section_name, {})
        parts.append(f"## {section.get('purpose', section_name)}")
        parts.append(section.get("content", ""))
        for b in section.get("bullets", []):
            parts.append(f"- {b}")
    # 6페이지 브리핑
    for page in final.get("pages", []):
        parts.append(f"### Step {page.get('step')}: {page.get('title')}")
        parts.append(page.get("content", ""))
    return "\n".join(parts)
