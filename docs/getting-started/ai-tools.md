# AI 도구 가이드

> 대상 독자: 전체 팀원
> 프로젝트에서 사용하는 AI 코딩 도구의 설정, 사용법, 주의사항을 정리합니다.

---

## 1. Claude Code (주요 도구)

Anthropic의 공식 CLI 도구. 터미널에서 직접 코드 작성, 리뷰, 배포까지 수행 가능.

### 프로젝트 컨텍스트

- **CLAUDE.md** 파일이 프로젝트 루트에 위치하며, Claude Code 실행 시 자동으로 로드됨
- 프로젝트 구조, 빌드 명령어, 코드 컨벤션, Git 규칙 등이 포함되어 있어 별도 설명 없이 컨텍스트 파악 가능
- CLAUDE.md는 버전 관리 대상 (git에 포함)

### 디렉토리 구조

```
.claude/
├── settings.json          # 팀 공유 설정 (git 추적)
├── settings.local.json    # 개인 설정 (.gitignore)
└── projects/              # 프로젝트별 메모리 (.gitignore)
```

### 커스텀 스킬

프로젝트에 등록된 슬래시 커맨드:

| 명령 | 설명 |
|------|------|
| `/deploy [frontend\|api\|all]` | Docker 빌드 + Hub 푸시 + deploy-test 배포 |
| `/test [unit\|backend\|integration\|e2e\|all]` | pytest/Playwright 테스트 실행 |
| `/migrate [upgrade\|current\|history\|revision\|downgrade]` | Alembic DB 마이그레이션 |
| `/seed [collect\|generate\|all]` | 데이터 파이프라인 실행 |

### 사용 팁

- `git worktree`와 조합하여 파트별 독립 세션 운영 가능
- 작업이 잘못되면 plan 모드로 복귀하여 재계획 (삽질 방지)
- 메인 컨텍스트는 전략/조율용, 구현 세부사항은 서브에이전트로 위임

## 2. Cursor

VS Code 기반 AI 에디터. 코드 편집과 AI 대화를 동시에 활용.

### 프로젝트 설정

- **`.cursorrules`** 파일로 프로젝트 컨텍스트 제공 (프로젝트 루트에 위치)
- Cursor가 코드 생성 시 참조할 규칙, 스택 정보, 컨벤션 등을 기술
- `.cursorrules`는 git에 포함하여 팀원 간 공유

### .gitignore 설정

```
.cursor/
```

- `.cursor/` 디렉토리는 개인 설정이므로 .gitignore에 포함

## 3. Codex / opencode / oh-my-opencode

OpenAI Codex CLI 또는 오픈소스 대안 도구.

### 프로젝트 컨텍스트

- 별도 설정 파일 불필요 — `docs/` 디렉토리와 `CLAUDE.md`를 참조하여 프로젝트 맥락 파악
- 필요 시 프롬프트에서 직접 파일 경로를 지정하여 컨텍스트 제공

### .gitignore 설정

```
.codex/
.opencode/
```

- 각 도구의 로컬 설정 디렉토리는 .gitignore에 포함

## 4. Entire CLI

체크포인트 기반 AI 코딩 도구.

### 주요 설정

- **`.entire/settings.json`** 에서 `manual-commit` 모드 설정 (자동 커밋 방지)
  ```json
  {
    "commit": "manual"
  }
  ```
- 자동 커밋이 활성화되면 AI 도구 흔적이 커밋에 포함될 수 있으므로, 반드시 수동 커밋 모드 사용

### 체크포인트 공유

- `entire/checkpoints/v1` 브랜치를 통해 체크포인트를 팀원 간 공유
- 체크포인트는 작업 상태 스냅샷으로, 롤백이나 참고용으로 활용

## 5. 공통 규칙

### AI 도구 흔적 금지

커밋 메시지, PR 본문에 AI 도구 사용 흔적을 남기지 않습니다.

**금지 항목:**
- `Co-Authored-By: Claude ...` 또는 유사 자동 삽입 태그
- `Generated with Claude Code`, `Created by Cursor` 등의 문구
- AI 도구 이름이 포함된 주석 (`// generated by ...`)

**이유:** AI 도구는 보조 수단이며, 최종 코드 품질의 책임은 커밋한 개발자에게 있음

### 코드 검토 필수

AI로 생성한 코드도 반드시 직접 검토 후 커밋합니다.

**체크리스트:**
- [ ] 생성된 코드가 프로젝트 컨벤션을 따르는지 확인
- [ ] 불필요한 주석이나 설명이 포함되지 않았는지 확인
- [ ] 테스트가 통과하는지 확인
- [ ] 기존 코드와의 일관성 확인

### 민감 정보 보호

- `.env` 파일, API 키, 비밀번호 등을 AI 도구에 직접 입력하지 않음
- AI 도구의 대화 로그에 민감 정보가 남지 않도록 주의
- `.env`는 .gitignore에 포함되어 있으나, AI 도구가 실수로 커밋하지 않도록 이중 확인
