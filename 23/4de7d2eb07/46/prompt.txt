### Step 3. deploy-test 미커밋 변경사항 처리 [중간]

먼저 변경 내용 확인:
```bash
ssh deploy-test "cd ~/adelie-investment && git diff docker-compose.prod.yml && git status --short"
```

**판단 기준**:
- docker-compose.prod.yml 변경이 의도적(포트/볼륨 튜닝 등)이면 → `git stash` 또는 커밋
- 실수/임시 변경이면 → `git checkout -- docker-compose.prod.yml`
- `output/` 폴더 → `.gitignore`에 추가하거나 삭제

처리 예시 (내용 확인 후 결정):
```bash
# 변경이 불필요하다면
ssh deploy-test "cd ~/adelie-investment && git checkout -- docker-compose.prod.yml && echo done"
```
