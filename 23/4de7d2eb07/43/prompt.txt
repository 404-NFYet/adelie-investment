## 핵심 갭 3가지

### Gap 1: max_tokens 부족 (근본 원인)

| 프롬프트 | 현재 max_tokens | 실패 빈도 | 증상 |
|----------|----------------|-----------|------|
| `narrative_body.md` | **5000** | 높음 | `summary` 섹션이 JSON 끝에 위치 → 잘림 |
| `hallucination_check.md` | **6000** | 중간 | validated_interface_2가 불완전 → fallback 트리거 |
| `3_tone_final.md` | **8000** | 중간 | 6페이지 전체 재작성 → Claude 2회 실패 → OpenAI fallback |
| `3_hallcheck_glossary.md` | **6000** | 낮음 | glossary 16개 이상시 잘림 |

**핵심**: `narrative_body`의 5000 토큰은 6개 섹션(background~summary) JSON 생성에 불충분.
`summary`가 JSON 마지막이라 가장 먼저 잘림 → fallback_validated도 `summary` 없음 → Pydantic 실패.

### Gap 2: fallback이 불완전한 원본을 그대로 사용

```
narrative_body (5000 토큰) → summary 잘림
      ↓
_build_unvalidated_interface2(narr) → 잘린 narrative 그대로 포함
      ↓
validate_interface2 fallback → fallback_validated에도 summary 없음
      ↓
방어 코드 → 누락 섹션에 {} 추가 + setdefault로 필드 채움 ← 이미 수정됨
      ↓
Pydantic → 빈 purpose="" 허용 → 통과
```

현재 로컬 코드에는 **누락 섹션 자동 생성** 코드가 추가되었으나 (`_REQUIRED_NARRATIVE_SECTIONS` 루프),
**근본 원인인 max_tokens 부족은 미해결**.

### Gap 3: glossary 아이템 내 null 필드 미처리

```
LLM glossary 생성 → 일부 아이템의 definition/domain = null
      ↓
assemble_output → glossary 리스트 자체 None→[] 처리만 있음
      ↓
Pydantic GlossaryItem(term: str, definition: str, domain: str) → 실패
```

현재 로컬 코드에는 **glossary 아이템 null→"" 변환** 코드가 추가되었으나 아직 배포 미완료.
