## 문제 1: 모바일 무한 새로고침

### 원인 분석

**1순위 (가장 유력): `handleAuthFailure()` 리다이렉트 루프**

`client.js:43-51` — 401 응답 시 `window.location.assign('/auth')` 호출 (전체 페이지 새로고침). 이 함수가 여러 곳에서 동시에 트리거될 수 있음:

- `UserContext.jsx:44` → `authApi.getMe(token)` → `authFetch` → 401 → refresh 실패 → `handleAuthFailure()`
- `PortfolioContext.jsx:86-88` → `loadData()` → `portfolioApi` → `authFetch` → 401 → `handleAuthFailure()`

여러 Context Provider가 동시에 `authFetch` 호출 → 각각 `handleAuthFailure()` 호출 → `window.location.assign('/auth')` 중복 호출 가능.

**2순위: Service Worker `autoUpdate` + `clientsClaim` 루프**

`vite.config.js:22,82` — 최근 배포가 여러 번 수행되면서, 모바일 브라우저의 Service Worker 캐시가 불안정해질 수 있음. `controllerchange` 이벤트 → `UpdateNotice.jsx`의 새로고침 유도.

### 수정 방안

#### Fix 1: `handleAuthFailure()` 중복 호출 방지

**파일**: `frontend/src/api/client.js`

```javascript
let isHandlingAuthFailure = false;

const handleAuthFailure = () => {
  if (isHandlingAuthFailure) return;
  isHandlingAuthFailure = true;
  localStorage.removeItem('token');
  localStorage.removeItem('authToken');
  localStorage.removeItem('refreshToken');
  window.dispatchEvent(new Event('auth:logout'));
  if (!window.location.pathname.startsWith('/auth')) {
    window.location.assign('/auth');
  }
};
```

#### Fix 2: `UserContext.getMe()` 실패 시 `handleAuthFailure` 대신 조용히 처리

**파일**: `frontend/src/contexts/UserContext.jsx`

`authApi.getMe()`를 `authFetch` 대신 일반 `fetch`로 호출하여, 401 시 `handleAuthFailure()`가 트리거되지 않도록 변경. 또는 `getMe` 전용 에러 핸들링으로 분리.

```javascript
// auth.js — getMe를 authFetch 대신 일반 fetch로
getMe: async (token) => {
  const response = await fetch(`${API_BASE_URL}/api/v1/auth/me`, {
    headers: token ? { Authorization: `Bearer ${token}` } : undefined,
  });
  if (!response.ok) throw new Error('Auth check failed');
  return response.json();
},
```

이렇게 하면 `getMe` 실패 시 `UserContext`의 `.catch()`에서 조용히 토큰 제거만 하고, 전체 페이지 새로고침 없이 Landing 페이지에 머무름.

### 수정 파일

| 파일 | 변경 |
|------|------|
| `frontend/src/api/client.js` | `handleAuthFailure()` 중복 호출 가드 추가 |
| `frontend/src/api/auth.js` | `getMe()`를 일반 `fetch`로 변경 (authFetch 제거) |
