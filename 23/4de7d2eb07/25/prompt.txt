### Phase B: 복습카드 시스템 (FlashCard)

#### B-FC1. 백엔드 — FlashCard 모델 + 마이그레이션
**파일:** `fastapi/app/models/flashcard.py` (신규 — 데모에서 포팅 + 확장)

```python
class FlashCard(Base):
    __tablename__ = "flashcards"
    id: Mapped[int] = mapped_column(primary_key=True)
    user_id: Mapped[int] = mapped_column(ForeignKey("users.id"))
    title: Mapped[str] = mapped_column(String(300))
    content_html: Mapped[str] = mapped_column(Text)  # 정리된 HTML
    key_points: Mapped[Optional[dict]] = mapped_column(JSONB, default=list)  # 핵심 포인트 배열
    topic_tags: Mapped[Optional[dict]] = mapped_column(JSONB, default=list)  # 주제 태그 배열
    source_session_id: Mapped[Optional[int]] = mapped_column(ForeignKey("tutor_sessions.id"))
    source_type: Mapped[str] = mapped_column(String(20), default="conversation")  # conversation|briefing|quiz
    created_at: Mapped[datetime] = mapped_column(default=datetime.utcnow)
```

**마이그레이션:** `database/alembic/versions/20260224_flashcards.py`

#### B-FC2. 백엔드 — FlashCard CRUD API
**파일:** `fastapi/app/api/routes/flashcards.py` (신규 — 데모에서 포팅)
- `POST /api/v1/flashcards` — 복습카드 생성
- `GET /api/v1/flashcards` — 목록 조회 (최신순, limit 파라미터)
- `DELETE /api/v1/flashcards/{card_id}` — 삭제
- JWT 인증 필수 (`get_current_user`)

#### B-FC3. 백엔드 — 복습카드 자동 생성 로직
**파일:** `fastapi/app/api/routes/tutor.py` (기존 파일에 추가)

`/복습` 슬래시 명령어 처리 시 또는 세션 저장 시:
1. 최근 대화 내용 수집 (session의 messages)
2. LLM 호출로 구조화된 복습카드 생성:
   - title: 대화 주제 요약 (1줄)
   - key_points: 핵심 포인트 3-5개
   - topic_tags: 관련 키워드 3-5개
   - content_html: 마크다운→HTML 정리 요약
3. FlashCard DB 저장

**프롬프트 설계:**
```
다음 대화를 학습 복습 카드로 정리해주세요.
- 제목: 1줄 요약
- 핵심 포인트: 3-5개 불릿
- 주제 태그: 3-5개 키워드
- 요약: 마크다운 형식, 200자 내외

대화 내용:
{conversation_text}
```
