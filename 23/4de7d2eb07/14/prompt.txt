# Dev Assistant v2 — 내부 AI 개발 어시스턴트 종합 설계

## Context

Adelie Investment 팀 내부 전용 AI 개발 어시스턴트를 **서브 프로젝트 수준**으로 구축한다.
기존 `feature/dashboard-chatbot` 챗봇(단순 Claude API 호출)을 넘어서, 코드베이스를 실시간 검색하고 서버 환경을 조회하며 코드를 실행할 수 있는 **에이전트 기반 시스템**을 설계한다.

### 핵심 설계 원칙
- **API 키 불필요**: 각 팀원의 Claude Code/Codex 구독을 그대로 사용 (MCP 서버 방식)
- **Dual Interface**: MCP 서버(primary) + Streamlit 웹 UI(secondary) — 같은 도구 레이어 공유
- **별도 인프라**: 전용 LXD 컨테이너 + 전용 git 브랜치로 격리 운영
- **계층적 에이전트**: Supervisor + 병렬 Worker 패턴 (LangGraph 1.0)
- **보안**: 네트워크 격리 + 공유 로그인 + 감사 로그 + 샌드박스 격리
- **사전 적재 지식**: 30+ 문서 파일(CLAUDE.md, docs/, .claude/rules/)을 DB + MCP Resource로 사전 인덱싱
- **실시간 코드 동기화**: git hook + 스케줄러로 코드 변경 자동 감지 + 증분 인덱싱
- **사용량 추적**: 팀원별 토큰/비용 대시보드 + Claude Code `/cost` 연동 가이드
- **Light/Dark UI**: Streamlit CSS 변수 기반 테마 전환, 양쪽 모드 시각 최적화

## 1. 아키텍처 개요

### 1.1 접속 경로

Dev Assistant는 기존 Streamlit 대시보드(`dashboard.adelie-invest.com`)의 새 페이지로 추가:

```
[외부] https://dashboard.adelie-invest.com/
    → Cloudflare Tunnel → deploy-test:8501 (nginx)
    → nginx → dashboard:8501 (Streamlit 컨테이너)
    → st.navigation() → pages/dev_assistant.py

※ 현재 대시보드는 인증 없이 외부 접근 가능 (보안 이슈)
→ 이번에 mutsa1234 공유 로그인을 대시보드 전체에 적용하여 해결
```

### 1.2 Dual-Interface 구조

```
┌─ MCP Clients (각자 본인 구독 사용) ─────────────────────────┐
│                                                              │
│  손영진: Claude Code (Pro) ──┐                               │
│  정지훈: Codex CLI ─────────┤  stdio / Streamable HTTP      │
│  허진서: Claude Code ───────┼──────→ MCP Server (:8090)     │
│  안례진: Codex CLI ─────────┤        (FastMCP)               │
│  도형준: Claude Code ───────┘                               │
│                                                              │
│  ※ LLM 호출은 각 팀원 계정에서 발생 → 팀 API 비용 0          │
└──────────────────────────────────────────────────────────────┘
                         ↕ 같은 도구 레이어 공유
┌─ Web UI (대시보드 확장) ─────────────────────────────────────┐
│  Streamlit Dev Assistant (:8502)                             │
│  → 빠른 조회/차트/세션 이력 관리                              │
│  → 공유 Claude API 키 사용 (기존 .env)                       │
└──────────────────────────────────────────────────────────────┘
                         ↕
┌─ Shared Tool Layer (devassistant 패키지) ────────────────────┐
│  Tools:  db_query | code_search | file_read | server_status  │
│          code_exec | deploy | git_ops | alembic_ops          │
│  RAG:    tree-sitter 청킹 + pgvector HNSW + RRF 하이브리드   │
│  Agent:  LangGraph Supervisor + 4 Worker (웹 UI용)           │
│  DB:     PostgreSQL (세션/메시지/코드인덱스/감사로그)          │
└──────────────────────────────────────────────────────────────┘
```

### 1.2 MCP가 API 키를 불필요하게 만드는 이유

```
[기존 방식]  대시보드 → Claude API (팀 공유 키, 팀이 비용 부담)
[MCP 방식]   개발자 → Claude Code (본인 구독) → MCP 서버 (도구만 제공)
             MCP 서버는 LLM을 호출하지 않음. 도구/데이터만 제공.
             LLM 호출은 각 팀원의 구독에서 처리됨.
```

Claude Code/Codex 모두 MCP 프로토콜을 지원하므로, **하나의 MCP 서버**로 양쪽 다 연결 가능.
한 번 `.mcp.json` 또는 `~/.codex/config.toml` 설정하면 이후 자동 연결.

## 3. 인증 & 보안 설계

### 3.1 통합 로그인 (웹 UI + MCP 연동)

**웹 UI**: 공유 계정 `mutsa1234` / `mutsa1234`로 로그인 → 본인 선택 → 세션 쿠키 유지

```python
# devassistant/config.py
SHARED_LOGIN = {"id": "mutsa1234", "pw_hash": bcrypt.hash("mutsa1234")}

TEAM_MEMBERS = {
    "dorae222":  {"name": "도형준", "role": "infra",    "icon": "🛠️", "server": "dev-hj"},
    "YJ99Son":   {"name": "손영진", "role": "frontend", "icon": "🎨", "server": "dev-yj99son"},
    "J2hoon10":  {"name": "정지훈", "role": "chatbot",  "icon": "🤖", "server": "dev-j2hoon10"},
    "jjjh02":    {"name": "허진서", "role": "backend",  "icon": "⚙️", "server": "dev-jjjh02"},
    "ryejinn":   {"name": "안례진", "role": "pipeline", "icon": "🧪", "server": "dev-ryejinn"},
}
```

**로그인 흐름**:
```
웹 UI (dashboard.adelie-invest.com 전체):
  mutsa1234 로그인 → 팀원 선택(아이콘 버튼) → 대시보드 전체 접근 허용
  ↓ (st.session_state에 user_name 저장 + 브라우저 쿠키 영속)
  이후 재접속 시 자동 로그인 (쿠키 유효 30일)
  ※ 대시보드 app.py에서 로그인 게이트 → 비인증 시 모든 페이지 차단

MCP:
  .mcp.json에 DEVASSIST_USER 설정 → 1회 설정 후 영속
  ↓ (Claude Code/Codex 세션마다 자동 연결)
  본인 MCP 계정 ↔ 웹 UI 계정 동일 user_name으로 세션 공유 가능
```

**핵심**: 로그인 후 본인이 파악되므로 Claude Code ↔ Codex ↔ 웹 UI 간 자유 이동 가능. 같은 `user_name`이면 대화 이력도 공유.

> **보안 개선**: 현재 `dashboard.adelie-invest.com`은 인증 없이 외부 접근 가능.
> `mutsa1234` 로그인을 `app.py` 최상단에 적용하여 **대시보드 전체**를 보호.

### 3.2 팀원별 MCP 설정 (1회 설정 후 영속)

**Claude Code 사용자** (프로젝트 루트 `.mcp.json`):
```json
{
  "mcpServers": {
    "adelie-dev": {
      "command": "python",
      "args": ["-m", "devassistant.mcp.server"],
      "cwd": "~/adelie-investment",
      "env": { "DEVASSIST_USER": "${GIT_USER_NAME}" }
    }
  }
}
```

**Codex CLI 사용자** (`~/.codex/config.toml`):
```toml
[mcp_servers.adelie-dev]
command = "python"
args = ["-m", "devassistant.mcp.server"]
env = { DEVASSIST_USER = "J2hoon10" }
startup_timeout_sec = 15
tool_timeout_sec = 120
```

**원격 접속 (외부 → 내부)** — SSH 터널 경유:
```bash
# 개인 머신에서 MCP 서버로 SSH 터널
ssh -L 8090:10.10.10.16:8090 deploy-test -N &
```

### 3.3 모델 선택 + 클라이언트 자유 이동

Claude Code/Codex 사용자는 본인 구독의 모델을 자유롭게 선택:
- Claude Code: `claude-sonnet-4-6`, `claude-opus-4-6`, `claude-haiku-4-5`
- Codex CLI: `gpt-4o`, `gpt-4o-mini`, `o3-mini`
- 웹 UI: 사이드바 드롭다운에서 모델 선택 (Claude API 키 사용)

MCP 서버는 모델에 무관 — 도구만 제공하므로 어떤 모델이든 호환.

**클라이언트 이동**: 같은 `user_name`으로 식별되므로:
- Claude Code에서 시작한 작업 → Codex CLI로 이어서 진행 가능
- MCP 세션 이력이 DB에 기록되므로 웹 UI에서도 조회 가능
- MCP 연결은 세션마다 새로 생성되지만, 서버 측 상태(DB)는 영속

### 3.4 보안 4계층

| 계층 | 대상 | 방법 |
|------|------|------|
| **네트워크** | MCP 서버 | 10.10.10.0/24 바인딩, 외부 접속은 SSH 터널 필수 |
| **인증** | 웹 UI | 공유 로그인(mutsa1234) + 팀원 선택, 쿠키 30일 유지 |
| **인증** | MCP | `DEVASSIST_USER` 환경변수, `.mcp.json` 영속 |
| **권한** | SQL/SSH | SELECT only, SSH 호스트 화이트리스트, 명령어 필터링 |
| **격리** | 코드 실행 | Docker --network=none, --read-only, mem 512M, timeout 30s |

## 4. PostgreSQL 스키마 (Alembic 마이그레이션)

기존 DB 확장 활용: `pgvector 0.8+`, `pg_trgm`, 신규 `ltree`

### 4.1 새 테이블 6개 (`narrative_invest` DB)

```sql
CREATE EXTENSION IF NOT EXISTS ltree;

-- 1. 세션 (대화 세션)
CREATE TABLE devassist_sessions (
    id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_name     VARCHAR(50) NOT NULL,
    title         VARCHAR(200),
    model_used    VARCHAR(100),
    interface     VARCHAR(20) DEFAULT 'mcp',  -- 'mcp' | 'web'
    metadata      JSONB DEFAULT '{}',
    created_at    TIMESTAMPTZ DEFAULT NOW(),
    updated_at    TIMESTAMPTZ DEFAULT NOW(),
    archived_at   TIMESTAMPTZ
);

-- 2. 메시지 (월별 파티셔닝)
CREATE TABLE devassist_messages (
    id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    session_id    UUID NOT NULL REFERENCES devassist_sessions(id) ON DELETE CASCADE,
    role          VARCHAR(20) NOT NULL,
    content       TEXT NOT NULL,
    tool_name     VARCHAR(100),
    tool_args     JSONB,
    tool_result   JSONB,
    tokens_used   JSONB,
    latency_ms    INTEGER,
    created_at    TIMESTAMPTZ DEFAULT NOW()
) PARTITION BY RANGE (created_at);
-- 월별 파티션 3개 자동 생성

-- 3. 코드베이스 인덱스 (RAG 보조 레이어 — 의미 기반 검색용)
--    Primary 코드 접근: SSH 직접 (항상 최신)
--    이 테이블: 벡터 검색 가속 레이어 (백그라운드 10분 갱신)
CREATE TABLE devassist_codebase_index (
    id            BIGSERIAL PRIMARY KEY,
    server_name   VARCHAR(50) NOT NULL DEFAULT 'shared',  -- 'dev-hj', 'dev-yj99son', 'shared'
    file_path     ltree NOT NULL,            -- 'fastapi.app.api.routes.briefing'
    file_path_raw TEXT NOT NULL,
    chunk_type    VARCHAR(30) NOT NULL,       -- 'function' | 'class' | 'method'
    chunk_name    VARCHAR(200),
    chunk_content TEXT NOT NULL,
    start_line    INTEGER NOT NULL,
    end_line      INTEGER NOT NULL,
    language      VARCHAR(20) DEFAULT 'python',
    git_hash      VARCHAR(40) NOT NULL,
    embedding     halfvec(768) NOT NULL,     -- HNSW + halfvec (50% 스토리지 절약)
    metadata      JSONB DEFAULT '{}',
    is_deleted    BOOLEAN DEFAULT FALSE,     -- soft delete (파일 삭제 시)
    indexed_at    TIMESTAMPTZ DEFAULT NOW(),
    UNIQUE(server_name, file_path_raw, chunk_type, chunk_name, start_line)
);

-- HNSW 벡터 인덱스 (IVFFlat 대비 2-10x 빠른 쿼리)
CREATE INDEX ix_codebase_hnsw ON devassist_codebase_index
    USING hnsw (embedding halfvec_cosine_ops)
    WITH (m=16, ef_construction=200);

-- ltree 경로 인덱스 (WHERE file_path @> 'fastapi.app.api' 지원)
CREATE INDEX ix_codebase_path ON devassist_codebase_index USING gist (file_path);

-- pg_trgm 퍼지 텍스트 검색 (코드 내용 + 심볼명)
CREATE INDEX ix_codebase_content_trgm ON devassist_codebase_index
    USING gin (chunk_content gin_trgm_ops);
CREATE INDEX ix_codebase_name_trgm ON devassist_codebase_index
    USING gin (chunk_name gin_trgm_ops);

-- 4. 에이전트 트레이스 (디버깅/분석)
CREATE TABLE devassist_agent_traces (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    session_id      UUID REFERENCES devassist_sessions(id) ON DELETE SET NULL,
    user_name       VARCHAR(50) NOT NULL,
    trace_type      VARCHAR(30) NOT NULL,
    node_name       VARCHAR(100),
    input_data      JSONB,
    output_data     JSONB,
    latency_ms      INTEGER,
    error_message   TEXT,
    langsmith_run_id VARCHAR(100),
    created_at      TIMESTAMPTZ DEFAULT NOW()
);

-- 5. 토큰 사용량 (비용 추적)
CREATE TABLE devassist_token_usage (
    id              BIGSERIAL PRIMARY KEY,
    user_name       VARCHAR(50) NOT NULL,
    model           VARCHAR(100) NOT NULL,
    provider        VARCHAR(30) NOT NULL,
    prompt_tokens   INTEGER DEFAULT 0,
    completion_tokens INTEGER DEFAULT 0,
    embedding_tokens INTEGER DEFAULT 0,
    cost_usd        NUMERIC(10,6) DEFAULT 0,
    context         VARCHAR(50),
    created_at      TIMESTAMPTZ DEFAULT NOW()
);

-- 6. 감사 로그 (보안)
CREATE TABLE devassist_audit_log (
    id            BIGSERIAL PRIMARY KEY,
    user_name     VARCHAR(50) NOT NULL,
    action        VARCHAR(100) NOT NULL,
    target        TEXT,
    result_status VARCHAR(20) DEFAULT 'success',
    ip_address    INET,
    details       JSONB DEFAULT '{}',
    created_at    TIMESTAMPTZ DEFAULT NOW()
);
```

### 4.2 설계 근거

| 선택 | 이유 |
|------|------|
| **halfvec(768)** | pgvector 0.7+ halfvec = 16bit float → 50% 스토리지 절감, 동일 recall |
| **HNSW** | IVFFlat 대비 30x 높은 QPS, O(log n) 검색, 빈 테이블에서도 작동 |
| **ltree** | 파일 경로 계층 쿼리(`@>` 연산자) → LIKE 패턴 대비 수십배 빠름 |
| **pg_trgm GIN** | 코드 식별자 퍼지 매칭 (함수명 오타 허용) |
| **월별 파티셔닝** | 메시지 테이블 partition pruning → 최근 조회 빠름 |
| **JSONB** | 도구 인자/결과, 메타데이터 유연 저장 |

## 5. Codebase RAG 파이프라인

### 5.1 파이프라인 흐름

```
[인덱싱]
Git 레포 → tree-sitter 파싱 → AST 기반 청킹 → 임베딩(768d) → PostgreSQL HNSW

[검색]
쿼리 → 임베딩 → ┬─ 벡터 유사도 (HNSW) ──→ top-50
                 └─ pg_trgm 퍼지 매칭 ───→ top-50
                           ↓
                    RRF 융합 (k=60) → top-20
                           ↓
                    리랭커 (Jina v2) → top-10
                           ↓
                    컨텍스트 빌더 → LLM
```

### 5.2 AST 청킹 (`rag/chunker.py`)

`tree-sitter-language-pack` (Python, JS, TS, SQL, YAML 160+ 언어 지원):

```python
# 청킹 전략
Python  → function_definition, class_definition, 메서드 개별 분리
JS/JSX  → function_declaration, arrow_function, class_declaration
SQL     → statement 단위 (CREATE TABLE, SELECT 등)
Markdown → ## 헤더 기반 섹션 분리
```

메타데이터 추출: `{imports, decorators, docstring, parent_class, sibling_signatures}`

### 5.3 하이브리드 검색 SQL (`rag/searcher.py`)

```sql
-- RAG 보조 검색 (2차). 1차는 SSH grep.
-- $5 = 사용자의 server_name (e.g., 'dev-hj')
-- 본인 서버 + shared 인덱스에서 벡터 검색
WITH vector AS (
    SELECT id, server_name, ROW_NUMBER() OVER (ORDER BY embedding <=> $1) AS rank
    FROM devassist_codebase_index
    WHERE server_name IN ('shared', $5) AND NOT is_deleted
      AND ($2::ltree IS NULL OR file_path @> $2)
    ORDER BY embedding <=> $1::halfvec(768) LIMIT 50
),
fuzzy AS (
    SELECT id, server_name, ROW_NUMBER() OVER (ORDER BY similarity(chunk_content, $3) DESC) AS rank
    FROM devassist_codebase_index
    WHERE server_name IN ('shared', $5) AND NOT is_deleted
      AND (chunk_content % $3 OR chunk_name % $3)
    LIMIT 50
),
rrf AS (
    SELECT id, server_name, 0.6/(60+rank) AS s FROM vector
    UNION ALL
    SELECT id, server_name, 0.4/(60+rank) AS s FROM fuzzy
)
SELECT id, SUM(s) + CASE WHEN server_name = $5 THEN 0.1 ELSE 0 END AS score
FROM rrf GROUP BY id, server_name ORDER BY score DESC LIMIT $4;
```

### 5.4 코드베이스 = 개인 LXD 서버 (실시간 SSH 접근)

**핵심 원칙**: 각 팀원의 LXD 서버가 곧 **그 팀원의 코드베이스**. 별도 복사본이 아닌, 서버 파일을 직접 실시간으로 접근.

```
┌─ 팀원이 질문 ────────────────────────────────────────────────────┐
│ "briefing 라우터 코드 보여줘"                                      │
└──────┬───────────────────────────────────────────────────────────┘
       ↓
┌─ 코드 접근 경로 ─────────────────────────────────────────────────┐
│                                                                    │
│  1. file_read(path)                                                │
│     → SSH 직접 접근: 본인 LXD 서버 파일 실시간 읽기                │
│     → 항상 최신 (uncommitted 변경 포함)                            │
│                                                                    │
│  2. code_search(query)                                             │
│     → [1차] SSH grep: 본인 LXD 서버에서 키워드 검색               │
│     → [2차] RAG 검색: 의미 기반 벡터 검색 (보조)                  │
│     → 결과 merge + 본인 서버 결과 우선                             │
│                                                                    │
│  3. git_ops(command)                                               │
│     → SSH 직접 실행: 본인 LXD 서버의 git log/diff/status          │
│                                                                    │
└──────────────────────────────────────────────────────────────────┘
```

**code_search 2단계 전략**:
```python
async def code_search(query: str, user_name: str, scope: str = None):
    host = TEAM_MEMBERS[user_name]["host"]

    # [1단계] SSH grep — 키워드 매칭 (빠름, 정확)
    grep_results = await ssh_cmd(host,
        f"cd {PROJECT_DIR} && grep -rn --include='*.py' --include='*.js' "
        f"--include='*.jsx' '{query}' {scope or '.'} | head -30"
    )

    # [2단계] RAG 벡터 검색 — 의미 기반 (유사 코드 탐색)
    rag_results = await hybrid_search(query, server_name=host)

    # merge: SSH grep 결과 우선, RAG로 보충
    return merge_and_rank(grep_results, rag_results)
```

**RAG 인덱스는 보조 레이어**:
- 직접 SSH 접근이 항상 primary (최신, uncommitted 포함)
- RAG 인덱스는 의미 기반 검색을 위한 가속 레이어 (백그라운드 갱신)
- 인덱스가 stale해도 무관 — SSH 결과가 항상 우선

**RAG 인덱스 갱신 (백그라운드, 10분 주기)**:
- 모든 LXD 서버 순회 → SSH로 변경 파일 감지 → 재인덱싱
- `server_name` 컬럼으로 서버별 인덱스 분리
- `ON CONFLICT (server_name, file_path_raw, chunk_type, chunk_name, start_line) DO UPDATE`
