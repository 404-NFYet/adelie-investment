## 강화 항목

### A. 시스템 프롬프트 (TUTOR_VIZ_SYSTEM_PROMPT)

**현재**: 3가지 차트 유형 (scatter, multi-line scatter, bar), ~450자
**변경**: 6가지 차트 유형 + 스켈레톤 + 7개 데이터 안전 규칙, ~2500자

#### 지원 차트 유형 (프론트엔드 `normalizeTraces.js` XY trace 호환 확인됨)

| # | 유형 | Plotly type | 용도 | 질문 패턴 |
|---|------|------------|------|-----------|
| 1 | 라인 차트 | scatter lines+markers | 종가 시계열 추이 | "추이", "흐름", "변화" |
| 2 | 세로 막대 | bar | 등락률/거래량 비교 | "등락률", "변동" |
| 3 | 가로 막대 | bar orientation h | 순위 나열 | "순위", "랭킹" |
| 4 | 그룹 막대 | 2+ bar traces, barmode group | 종목간 비교 | "비교", "vs" |
| 5 | 이중축 혼합 | scatter(주가) + bar(거래량), yaxis2 | 가격+거래량 동시 | "거래량과 함께" |
| 6 | 영역 scatter | scatter fill tozeroy + annotations | 추이+마일스톤 | "전환점", "급등" |

#### 금지 유형 (프론트엔드 normalizeTraces에 핸들러 없음)
`candlestick`, `waterfall`, `sankey`, `scatterpolar`, `choropleth`, `pie`

#### 스타일 규칙 (datapipeline + 프론트엔드 정합)
- 색상: `["#FF6B35", "#004E89", "#1A936F", "#C5D86D", "#8B95A1"]`
- 등락률 bar: 양수 `#FF6B35`, 음수 `#004E89`
- 배경: 투명 (`rgba(0,0,0,0)`)
- 폰트: `IBM Plex Sans KR`, size 12
- 마진: `{t:40, b:50, l:55, r:25}` (모바일 480px)
- annotation: `showarrow:true, ax:0, ay:-30`
- 날짜: `YYYY-MM-DD` 또는 `MM/DD`

#### 데이터 안전 규칙 (7개, `3_hallcheck_chart.md` 반영)
1. 가상 데이터 절대 금지
2. 추정치 라벨 금지 (`(E)`, `Est`, `추정`, `예상`, `mock`)
3. x/y 배열 길이 동일 필수
4. 모든 수치 = chart_data 원본과 정확 일치
5. 데이터 3개 미만 → 빈 차트
6. 축 절단 금지 (y축 range가 데이터 전체 포함)
7. chart_data에 있는 날짜만 사용

### C. 검증 로직 (_validate_chart)

**현재**: 2개 gate (포인트 수, 추정 마커)
**변경**: 7개 gate — datapipeline `chart_agent.py` + `3_hallcheck_chart.md` 수준

| Gate | 검증 | 참고 |
|------|------|------|
| 1 | 유효 데이터 포인트 ≥ 3 | 기존 유지 |
| 2 | 추정치 마커 감지 (trace data + layout + annotations 전체) | `chart_agent.py:_contains_estimation_marker` 포팅, 기존은 layout만 검사 |
| 3 | x/y 배열 길이 동일 | `3_chart_generation.md` Rule #3 |
| 4 | 금지 차트 유형 (candlestick 등) | 프론트엔드 normalizeTraces 분석 기반 |
| 5 | 날짜 정합성 (chart_data 날짜만 허용) | 신규 |
| 6 | 수치 범위 교차 검증 (원본 데이터 ±10% 내) | `3_hallcheck_chart.md` "데이터 정확성" |
| 7 | 축 절단 경고 (y range vs 데이터 범위) | `3_hallcheck_chart.md` "시각적 오해 소지" |

#### 신규 헬퍼 함수

