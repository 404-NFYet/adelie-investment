## 구조 이해 (중요)

```yaml
# docker-compose.dev.yml
frontend:
  image: dorae222/adelie-frontend-dev:latest
  build:                          # ← build + image 병기 → up --build 시 latest 덮어씀
    context: ./frontend
    dockerfile: Dockerfile.dev
  volumes:
    - ./frontend/src:/app/src     # ← 소스코드는 LXD 서버 로컬 checkout에서 서빙
    - ./frontend/public:/app/public
    - ./frontend/index.html:/app/index.html
```

- Docker 이미지 역할: `node_modules` + Vite 런타임만 제공
- 실제 서빙 소스: 각 LXD 서버의 로컬 git checkout (`dev-final/*`)
- **이미지 교체만으로는 UI 소스코드 미변경** (volume mount가 덮어씀)

### Step 2. docker-compose.dev.yml 개선

**변경 사항:**
1. `image:` 태그를 `feb20-stable`로 명시 (latest 대신)
2. `build:` 섹션 **제거** — 실수로 `--build` 실행해도 버전 고정 이미지 안전
3. Vite dev 모드 + HMR + volume mount 완전 유지

```yaml
# 변경 전
frontend:
  image: dorae222/adelie-frontend-dev:latest
  build:
    context: ./frontend
    dockerfile: Dockerfile.dev
  ports:
    - "3001:3001"
  volumes:
    - ./frontend/src:/app/src
    - ./frontend/public:/app/public
    - ./frontend/index.html:/app/index.html
  environment:
    - PROXY_TARGET=http://backend-api:8082

# 변경 후
frontend:
  image: dorae222/adelie-frontend-dev:feb20-stable    # ← 명시적 버전 태그
  # build: 제거 — 이미지는 별도로 배포, 실수 빌드 방지
  ports:
    - "3001:3001"
  volumes:
    - ./frontend/src:/app/src
    - ./frontend/public:/app/public
    - ./frontend/index.html:/app/index.html
  environment:
    - PROXY_TARGET=http://backend-api:8082
```
