## 실행 계획

### Step 1: dev-final/* 브랜치 생성 (release/feb20-stable + dev/* 머지)

**파일**: `lxd/Makefile`에 `dev-final-branches` 타겟 추가

실행 로직:
```bash
CURRENT=$(git branch --show-current)

# 각 파트별 브랜치 생성 + dev/* 머지
declare -A PARTS=(
  [frontend]=dev/frontend
  [chatbot]=dev/chatbot
  [backend]=dev/backend
  [pipeline]=dev/pipeline
  [infra]=dev/infra
)

for part in "${!PARTS[@]}"; do
  git checkout release/feb20-stable
  git checkout -b dev-final/$part 2>/dev/null || git checkout dev-final/$part
  git fetch origin
  git merge --no-edit origin/${PARTS[$part]} && \
    git push origin dev-final/$part && \
    echo "  ✅ dev-final/$part" || \
    echo "  ⚠️ dev-final/$part: 충돌 발생 — 수동 해결 필요"
done

git checkout $CURRENT  # 원래 브랜치 복귀
```

> 충돌 발생 시 해당 브랜치에서 수동 해결 후 `git push origin dev-final/$part`

### Step 2: 각 LXD 서버에서 dev-final/* checkout

**파일**: `lxd/Makefile`에 `dev-final-checkout` 타겟 추가

실행 로직 (서버당):
```bash
lxc exec dev-yj99son -- bash -c "
  cd /home/ubuntu/adelie-investment
  git fetch origin
  git stash 2>/dev/null || true   # 미커밋 변경사항 임시 보관
  git checkout dev-final/frontend
  echo '✅ dev-yj99son: dev-final/frontend'
"
```

각 서버 → 각 브랜치 매핑으로 동일하게 처리.

### Step 3: Docker 이미지 재빌드 + 서비스 재시작

**파일**: `lxd/Makefile`에 `dev-final-rebuild` 타겟 추가

docker-compose.dev.yml의 frontend/backend-api/ai-pipeline 모두 로컬 빌드 대상이므로
feb20-stable 코드 기반으로 전체 rebuild 필요:
```bash
lxc exec dev-{name} -- bash -c "
  cd /home/ubuntu/adelie-investment
  docker compose -f docker-compose.dev.yml build --no-cache frontend backend-api 2>&1 | tail -5
  docker compose -f docker-compose.dev.yml up -d --remove-orphans
"
```

> ai-pipeline은 개발 중 상시 실행 필요 없으므로 frontend + backend-api만 rebuild.
> ai-pipeline은 필요 시 수동 빌드.

### Step 4: DB sync (사용자 신호 대기 후 별도 실행)

파이프라인 수집 완료 알림을 받은 후 실행:
```bash
bash database/scripts/sync_dev_data.sh
# 또는
make sync-dev-data
```

> ⏳ **Step 1~3은 즉시 실행, Step 4는 사용자 알림 후 별도 실행**

## Makefile 타겟 설계 (`lxd/Makefile`)

```makefile
# dev-final/* 브랜치 생성 (release/feb20-stable + dev/* 머지)
dev-final-branches:
	@echo "=== dev-final/* 브랜치 생성 (feb20-stable + dev/* 머지) ==="
	@cd $(ROOT) && CURRENT=$$(git branch --show-current); \
	git fetch origin; \
	for entry in "frontend:dev/frontend" "chatbot:dev/chatbot" "backend:dev/backend" "pipeline:dev/pipeline" "infra:dev/infra"; do \
		part=$${entry%%:*}; src=$${entry##*:}; \
		git checkout release/feb20-stable; \
		git checkout -b dev-final/$$part 2>/dev/null || git checkout dev-final/$$part; \
		git merge --no-edit origin/$$src \
			&& git push origin dev-final/$$part \
			&& echo "  ✅ dev-final/$$part (← $$src)" \
			|| echo "  ⚠️ dev-final/$$part: 충돌 발생 — 수동 해결 필요"; \
	done; \
	git checkout $$CURRENT
	@echo "=== 브랜치 생성 완료 ==="

# 각 LXD 서버에서 dev-final/* checkout
dev-final-checkout:
    @echo "=== 각 LXD 서버 dev-final/* checkout ==="
    @lxc exec dev-yj99son  -- bash -c "cd /home/ubuntu/adelie-investment && git fetch origin && git stash 2>/dev/null; git checkout dev-final/frontend  && echo '  ✅ dev-yj99son: dev-final/frontend'"  2>/dev/null || echo "  ❌ dev-yj99son"
    @lxc exec dev-j2hoon10 -- bash -c "cd /home/ubuntu/adelie-investment && git fetch origin && git stash 2>/dev/null; git checkout dev-final/chatbot   && echo '  ✅ dev-j2hoon10: dev-final/chatbot'"  2>/dev/null || echo "  ❌ dev-j2hoon10"
    @lxc exec dev-jjjh02   -- bash -c "cd /home/ubuntu/adelie-investment && git fetch origin && git stash 2>/dev/null; git checkout dev-final/backend  && echo '  ✅ dev-jjjh02: dev-final/backend'"   2>/dev/null || echo "  ❌ dev-jjjh02"
    @lxc exec dev-ryejinn  -- bash -c "cd /home/ubuntu/adelie-investment && git fetch origin && git stash 2>/dev/null; git checkout dev-final/pipeline  && echo '  ✅ dev-ryejinn: dev-final/pipeline'"  2>/dev/null || echo "  ❌ dev-ryejinn"
    @lxc exec dev-hj       -- bash -c "cd /home/ubuntu/adelie-investment && git fetch origin && git stash 2>/dev/null; git checkout dev-final/infra     && echo '  ✅ dev-hj: dev-final/infra'"        2>/dev/null || echo "  ❌ dev-hj"
    @echo "=== checkout 완료 ==="

# 각 LXD 서버 Docker 재빌드 + up -d
dev-final-rebuild:
    @echo "=== 각 LXD 서버 Docker 재빌드 ==="
    @for srv in dev-yj99son dev-j2hoon10 dev-jjjh02 dev-ryejinn dev-hj; do \
        echo "  📦 $$srv 빌드 중..."; \
        lxc exec $$srv -- bash -c " \
            cd /home/ubuntu/adelie-investment && \
            docker compose -f docker-compose.dev.yml build frontend backend-api 2>&1 | tail -3 && \
            docker compose -f docker-compose.dev.yml up -d --remove-orphans" \
            && echo "  ✅ $$srv 재시작 완료" || echo "  ❌ $$srv 빌드 실패"; \
    done
    @echo "=== 재빌드 완료 ==="

# dev-final 전체 셋업 (브랜치 생성 → checkout → 재빌드)
dev-final-setup: dev-final-branches dev-final-checkout dev-final-rebuild
    @echo "✅ dev-final 전체 셋업 완료"
```

## 리스크 및 주의사항

- **git stash로 인한 변경사항 보관**: 각 서버에 미커밋 변경이 있으면 stash에 보관됨. 팀원에게 `git stash pop` 안내 필요.
- **Docker 빌드 시간**: 각 서버에서 병렬 실행되지만 순차이므로 5서버 × ~3분 ≒ 15분 소요 가능. `dev-final-rebuild`를 개별로 실행하면 빠름.
- **DB sync 타이밍**: 파이프라인 수집 완료 후 sync해야 최신 데이터 반영. Discord 알림 확인 또는 `docker logs`로 완료 확인.
- **담당자 커밋**: `dev-final-branches` 타겟이 브랜치를 push할 때 현재 git config(dorae222)로 진행. 브랜치 생성 push는 커밋이 없으므로 무방.
