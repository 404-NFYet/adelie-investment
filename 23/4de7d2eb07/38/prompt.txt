### Step 3: 팀원별 개인 로컬 DB 구성 + 프로덕션 데이터 복제

#### 아키텍처 변경

```
기존: 모든 LXD 서버 → infra-server 공유 DB (10.10.10.10:5432) [공유 → 충돌 위험]

신규:
deploy-test prod DB
    ↓  pg_dump + lxc exec pipe (서버별 1회 sync)
  dev-yj99son 로컬 postgres (postgres:5432 docker network)
  dev-j2hoon10 로컬 postgres
  dev-jjjh02 로컬 postgres
  dev-ryejinn 로컬 postgres
  dev-hj 로컬 postgres
      ↑ 각자 개인 격리 DB
```

- `docker-compose.dev.yml`의 backend-api `environment:` 섹션이 이미 `DATABASE_URL: postgres:5432` (로컬 postgres) 명시
- `.env`의 `DATABASE_URL=10.10.10.10:5432`는 compose `environment:`에 의해 override → 코드 변경 없음
- 팀원이 직접 데이터 망가뜨려도 다른 팀원에게 영향 없음

#### 3a. 각 LXD 서버 로컬 postgres 기동 + 마이그레이션

```bash
for srv in dev-yj99son dev-j2hoon10 dev-jjjh02 dev-ryejinn dev-hj; do
  lxc exec $srv -- bash -c "
    cd /home/ubuntu/adelie-investment
    # 로컬 postgres 기동
    docker compose -f docker-compose.dev.yml up -d postgres
    sleep 6
    # alembic HEAD까지 마이그레이션 (backend-api 컨테이너가 postgres에 연결)
    docker compose -f docker-compose.dev.yml run --rm backend-api \
      sh -c 'cd /app/database && alembic upgrade head'
    echo '  ✅ $srv 마이그레이션 완료'
  " 2>/dev/null || echo "  ❌ $srv 실패"
done
```

#### 3b. deploy-test → 각 LXD 로컬 postgres 데이터 복제

deploy-test에서 pg_dump → lxc exec로 각 서버 로컬 postgres에 직접 pipe:

```bash
# deploy-test에서 콘텐츠 테이블 덤프 생성 (1회)
ssh deploy-test "docker exec adelie-postgres pg_dump \
  -U narative -d narrative_invest \
  --data-only --disable-triggers \
  -t stock_listings -t daily_briefings -t briefing_stocks \
  -t historical_cases -t case_matches -t case_stock_relations \
  -t broker_reports" > /tmp/adelie_prod_content.sql

# 각 LXD 서버 로컬 postgres에 복제
for srv in dev-yj99son dev-j2hoon10 dev-jjjh02 dev-ryejinn dev-hj; do
  echo "  → $srv 데이터 복제 중..."
  lxc exec $srv -- bash -c "
    docker exec -i adelie-investment-postgres-1 \
      psql -U narative -d narrative_invest \
      -c 'SET session_replication_role = replica;' 2>/dev/null
    docker exec -i adelie-investment-postgres-1 \
      psql -U narative -d narrative_invest
  " < /tmp/adelie_prod_content.sql \
  && echo "  ✅ $srv 완료" || echo "  ❌ $srv 실패"
done

rm -f /tmp/adelie_prod_content.sql
```

#### 3d. lxd/Makefile에 `dev-local-db-setup` 타겟 추가

```makefile
# 팀원별 개인 로컬 DB 초기화 + deploy-test 프로덕션 데이터 복제
dev-local-db-setup:
	@echo "=== Step 1: 로컬 postgres 기동 + 마이그레이션 ==="
	@for srv in dev-yj99son dev-j2hoon10 dev-jjjh02 dev-ryejinn dev-hj; do \
		echo "  → $$srv 기동 중..."; \
		lxc exec $$srv -- bash -c " \
			cd /home/ubuntu/adelie-investment && \
			docker compose -f docker-compose.dev.yml up -d postgres && sleep 6 && \
			docker compose -f docker-compose.dev.yml run --rm backend-api \
				sh -c 'cd /app/database && alembic upgrade head'" \
			&& echo "  ✅ $$srv" || echo "  ❌ $$srv"; \
	done
	@echo ""
	@echo "=== Step 2: deploy-test → 각 서버 로컬 DB 데이터 복제 ==="
	@SSH_HOST=deploy-test; \
	DUMP_FILE=/tmp/adelie_prod_content_$$(date +%Y%m%d%H%M).sql; \
	ssh $$SSH_HOST "docker exec adelie-postgres pg_dump \
		-U narative -d narrative_invest \
		--data-only --disable-triggers \
		-t stock_listings -t daily_briefings -t briefing_stocks \
		-t historical_cases -t case_matches -t case_stock_relations \
		-t broker_reports" > $$DUMP_FILE; \
	for srv in dev-yj99son dev-j2hoon10 dev-jjjh02 dev-ryejinn dev-hj; do \
		echo "  → $$srv 데이터 복제 중..."; \
		lxc exec $$srv -- bash -c " \
			docker exec -i adelie-investment-postgres-1 \
				psql -U narative -d narrative_invest \
				-c 'SET session_replication_role = replica; \
				    TRUNCATE TABLE stock_listings, daily_briefings, briefing_stocks, \
				    historical_cases, case_matches, case_stock_relations, broker_reports \
				    CASCADE; \
				    SET session_replication_role = DEFAULT;'" < /dev/null; \
		" < $$DUMP_FILE \
		&& echo "  ✅ $$srv" || echo "  ❌ $$srv"; \
	done; \
	rm -f $$DUMP_FILE
	@echo ""
	@echo "=== Step 3: backend-api 재시작 ==="
	@for srv in dev-yj99son dev-j2hoon10 dev-jjjh02 dev-ryejinn dev-hj; do \
		lxc exec $$srv -- bash -c " \
			cd /home/ubuntu/adelie-investment && \
			docker compose -f docker-compose.dev.yml restart backend-api" \
			&& echo "  ✅ $$srv" || echo "  ❌ $$srv"; \
	done
	@echo "✅ dev-local-db-setup 완료 (각 팀원 개인 로컬 DB 활성)"
```

## 리스크 및 주의사항

- **마이그레이션 실행 중 다운타임 없음**: `alembic upgrade head`는 ADD COLUMN만이므로 서비스 중단 불필요
- **frontend/backend 이미지 태그 통일**: `feb20-stable` + `latest` 혼용 상태 해소 필요
- **로컬 DB sync 시간**: 서버당 pg_dump/restore 포함 ~2분 예상
- **infra-server 공유 DB 역할 종료**: 각 LXD가 개인 로컬 postgres 사용으로 전환. infra-server tmp-postgres-1은 계속 실행 중이지만 더 이상 LXD dev 환경 연결 주소로 사용 안 함
- **docker-compose environment 우선순위**: `environment:` 섹션이 `env_file:` 보다 우선 → `.env`의 `DATABASE_URL=10.10.10.10:5432` 자동 override, 코드 변경 불필요
- **develop 브랜치 안정성**: Step 2에서 develop으로 전환 시 최신 커밋 상태 확인 필요
