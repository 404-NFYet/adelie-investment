## Context

- `main` push → 프로덕션(deploy-test) 자동 배포

## 1. 브랜치 전략

```
main                    ─── 소개/README용 (CI/CD 없음)
  └── develop           ─── staging + 프로덕션 배포 (deploy-test, :latest 태그)
        └── dev-final/* ─── LXD 개인 개발 서버 자동 배포
              └── feature/* ─── 개인 피처 작업 (CI만)
```

**핵심 결정:** `main`은 소개용으로 유지, `develop`이 실질적인 배포 브랜치.
현재 `deploy.yml`의 `develop → deploy-test` 흐름을 그대로 계승하되, `dev-final/*` → LXD 자동화를 추가한다.

### 브랜치별 규칙

| 브랜치 | merge 방법 | 리뷰 | CI | 배포 |
|--------|-----------|------|-----|------|
| `main` | PR from develop (선택) | — | — | ❌ (소개용) |
| `develop` | PR from dev-final/* | 불필요 | 통과 필수 | ✅ deploy-test (:latest) + Discord |
| `dev-final/*` | 직접 push 허용 | 없음 | 실행 (non-blocking) | ✅ 해당 LXD 서버 |
| `feature/*` | PR to dev-final/* | 없음 | 실행 | ❌ |

### 일회성 정리 작업
- `develop` → 각 `dev-final/*` merge (히스토리 통합, LXD 변경사항 보존 확인 후)
- `dev/*` 브랜치 전원 삭제 (dev-final/*로 대체됨)
- `release/feb20-stable` → 태그 `v1.0-stable`로 전환 후 브랜치 삭제

## 3. Makefile 재설계

### Root `Makefile` — 팀원 공통 명령어

섹션 구분:
```
=== 개발 환경 (팀원 공통) ===
make dev          docker-compose.dev.yml 전체 스택 실행
make dev-down     개발 환경 중지
make status       컨테이너 상태 + 현재 브랜치 출력

=== 테스트 ===
make test         전체 테스트 (pytest + 빌드 확인)
make test-unit    유닛 테스트만
make test-e2e     Playwright E2E

=== DB ===
make migrate      alembic upgrade heads
make db-reset     DB 초기화 (개발 환경 전용)

=== 빌드/배포 (인프라 전용 — main 브랜치에서만) ===
make build        전체 Docker 이미지 빌드
make push         Docker Hub push (REGISTRY/TAG 변수)
make deploy       docker-compose.prod.yml up -d (deploy-test에서)
make rollback     :stable 이미지로 롤백

=== 유틸 ===
make clean        Docker 캐시 + __pycache__ 정리
make logs         프로덕션 로그 tail
```

변경점:
- AWS 관련 타겟 제거 (없음)
- `dev-frontend-local`, `dev-api-local` 등 세분화된 타겟 유지
- `sync-dev-data` → `db-sync` 별칭 추가
- `make help`가 섹션별로 정렬되어 출력

### `lxd/Makefile` — 인프라 전용

섹션 구분:
```
=== LXD 서버 관리 ===
health            5대 서버 브랜치/컨테이너 상태 한눈에 보기
sync              각 LXD 서버 git pull + docker compose up
db-sync           prod DB → 각 서버 로컬 dev DB 복제
jwt-fix           JWT_SECRET 기본값 서버 자동 갱신
git-check         git user/email/remote 설정 점검

=== Docker Hub 배포 ===
push              이미지 빌드 + Docker Hub push
deploy            deploy-test 배포 (git pull + compose up)
rollback          :stable 이미지로 deploy-test 롤백

=== dev-final 환경 ===
dev-final-setup   브랜치 생성 + LXD checkout + 재빌드 (전체)
```

### Step B. 팀원 공지 (Discord) 후 각 서버 stash 확인

```bash
# 각 LXD 서버에서 미커밋 변경사항 자동 stash 후 pull
for srv in dev-yj99son dev-j2hoon10 dev-jjjh02 dev-ryejinn dev-hj; do
  lxc exec $srv -- su - ubuntu -c "
    cd ~/adelie-investment
    git stash push -m 'auto-stash before branch sync $(date +%Y%m%d)' 2>/dev/null || true
    git pull origin \$(git branch --show-current)
    echo '✅ '\$(git branch --show-current)
  "
done
```

### Step C. develop → dev-final/* 히스토리 통합

```bash
# HOST에서 각 dev-final/* 브랜치에 develop merge
for branch in dev-final/frontend dev-final/chatbot dev-final/backend dev-final/pipeline dev-final/infra; do
  git checkout $branch
  git merge origin/develop --no-edit 2>/dev/null || {
    echo "⚠️  $branch 충돌 — 수동 해결 필요"
    git merge --abort
    continue
  }
  git push origin $branch
  echo "✅ $branch"
done
```

### Step D. 각 LXD 서버 pull + stash pop (작업 복구)

```bash
for srv in dev-yj99son dev-j2hoon10 dev-jjjh02 dev-ryejinn dev-hj; do
  lxc exec $srv -- su - ubuntu -c "
    cd ~/adelie-investment
    git pull origin \$(git branch --show-current)
    git stash pop 2>/dev/null && echo '  stash 복구 완료' || echo '  stash 없음'

done
```
