# Phase 3: 페이지 전환 + 자동 실행 확인

## [3A] 네비게이션 액션 시 확인 모달

챗봇이 페이지 전환 액션을 제안할 때, 사용자에게 확인:

> "포트폴리오 페이지로 이동할까요?"
> [이동하기] [자동 실행 모드 켜기] [취소]

**"자동 실행 모드"**: 이후 네비게이션/툴 액션을 확인 없이 자동 실행.

**파일**: `frontend/src/hooks/useAgentControlOrchestrator.js`

```javascript
const [autoMode, setAutoMode] = useState(false);

const executeAction = async (action) => {
  // 자동 모드 또는 low-risk 액션 → 즉시 실행
  if (autoMode || action.risk === 'low') {
    return await _doExecute(action);
  }
  // high-risk (매수/매도) 또는 navigate → 확인 요청
  const confirmed = await showConfirmDialog(action);
  if (confirmed === 'auto') setAutoMode(true);
  if (confirmed) return await _doExecute(action);
};
```

**새 컴포넌트**: `frontend/src/components/agent/ActionConfirmDialog.jsx`
- 3버튼: 실행 / 자동 모드 / 취소
- 자동 모드는 세션 단위 (새 세션 시 리셋)

# Phase 4: 복습 카드 시스템

## 설계 원칙

- **저장 버튼** 누름 → 복습 카드 생성 (리치 메타데이터 + 요약)
- **저장 안 함** → 대화 기록만 (히스토리 목록에서 클릭 시 세션 재개)
- 교육탭: 복습 카드가 기존 기사/브리핑 카드 대체
- 복습 카드 클릭 → 정적 카드 뷰 (요약 + 핵심 포인트 + 차트)
- 대화 기록 클릭 → 세션 재개 (대화 계속)

## [4A] 복습 카드 메타데이터 강화

**저장(핀) 시 리치 메타데이터 수집**:

**파일**: `fastapi/app/api/routes/tutor_sessions.py` — pin 엔드포인트 확장

핀 시 LLM으로 대화 요약 + 핵심 포인트 추출:
```python
@router.post("/sessions/{session_id}/pin")
async def pin_session(...):
    # 기존 pin 로직
    session.is_pinned = True
    session.pinned_at = datetime.now(KST)

    # 복습 카드 메타데이터 생성
    messages = await _get_session_messages(db, session.id)
    review_meta = await _generate_review_card_meta(messages)
    session.review_summary = review_meta["summary"]        # 3줄 요약
    session.review_key_points = review_meta["key_points"]  # 핵심 포인트 리스트
    session.review_topics = review_meta["topics"]           # 다룬 주제/종목
```

**마이그레이션**: `TutorSession`에 컬럼 추가:
- `review_summary` (Text)
- `review_key_points` (JSON, list[str])
- `review_topics` (JSON, list[str])
